vCardLib Documentation


In v2.2.0 breaking changes have been introduced (for good reason). See the v2.2.0 changelog

vCardLib is a .NET library written entirely in C#. It is designed to enable developers read contacts stored in a vCard file (*.vcf or *.vcard). As at today (15th March 2017), the latest version of the library (v2.2.0) supports vCard files of version 2.1 and 3.0 only. The library has its classes under the root namespace vCardLib. The class 'vCard.Deserializers' has a class Deserializer which has static methods FromFile and FromStreamReader which as their names imply allow you extract contacts from a file and a StreamReader object respectively.

The FromFile and FromStreamReader methods return a vCardCollection object which inherits from CollectionBase. The vCardCollection object implements IEnumerable and can be iterated over using the foreach operator. Each object is of the vCard type from which properties can be read.

How to use the library

To install this library from Nuget you can click the "Nuget" button above. If you want to install it via the Package Manager, you can run the command:
Install-Package vCardLib.dll

To read contacts

To make the deserialization methods and classes of the library available, add this at the top of your *.cs file: 
        
          using vCardLib;
          using vCardLib.Deserializers;
          using vCardLib.Collections;

In your class you call the static method Deserializer.FromFile and pass a string containing the path to a vcf or vcard file to it: 
        
          string filePath = //path to vcf or vcard file;
          vCardCollection contacts = Deserializer.FromFile(filePath);
Or pass a StreamReader object to it: 
        
          StreamReader sr = //generate a streamreader somehow;
          vCardCollection contacts = Deserializer.FromStreamReader(sr);
Iterate over the collection and pick the vCard objects: 
        
          foreach(vCard contact in contacts)
          {
            Console.WriteLine(contact.FormattedName);
          }

To write/save contacts

To make the serialization methods and classes of the library available, add this at the top of your *.cs file: 
        
          using vCardLib;
          using vCardLib.Serializers;
          using vCardLib.Collections;
          using vCardLib.Helpers;

In your class you call the static method Serializer.Serialize and pass a non-null vCard object, a string containing the path to a vcf or vcard file, a Version version indicating in what format the vcard should be serialized and optionally a WriteOptions option telling the method whether to overwrite if the file exists to it: 
        
          string filePath = //path to vcf or vcard file that you want created or overwritten;
          vCard vcard = //create one somehow;
          bool status = Serializer.Serialize(vcard, filePath, Version.V3, WriteOptions.Overwrite);
          //check if the serialization was successful
          if (status)
          {
            //successful
          }
          else
          {
            //unsuccesful
          }
Or serialize a vCardCollection object: 
        
          string filePath = //path to vcf or vcard file that you want created or overwritten;
          vCardCollection collection = new vCardCollection();
          //insert vcards into the collection
          bool status = Serializer.Serialize(collection, filePath, Version.V2, WriteOptions.ThrowError);
          //check if the serialization was successful
          if (status)
          {
            //successful
          }
          else
          {
            //unsuccesful
          }

Library Version Changelogs

Library Version 2.2.1

v2.2.1 was written with performance and resource usage a major priority. String handling was particularly dealt with in the most resource safe way. Also one new property was added, NOTE property is now supported.

  1. Note: string

Library Version 2.2.0

With the release of version 2.2.0 of this library, few properties were added and some breaking changes were introduced. The following properties were removed:

  1. Surname: string
  2. Middlename: string
  3. Firstname: string
To be replaced with these in order to keep conformance with the RFC spec:
  1. FamilyName: string
  2. GivenName: string
  3. MiddleName: string
  4. Prefix: string
  5. Suffix: string
The URL property was renamed to Url to conform with C# naming conventions.

The following properties changed types:
  1. Version: Version (V2, V3, V4) (is now of enum type from float)
  2. BirthDay: DateTime? (is now a nullable DateTime object from a DateTime object)
The following properties were added:
  1. Revision: DateTime?
  2. Geo: Geo
    • Longitude: double
    • Latitude: double
  3. XSkypePstnNumber: string
  4. XSkypeDisplayName: string

Library Version 2.1.0

With the release of version 2.1.0 of this library, support has been added for writing of vCard objects to vcf files. It supports writing one file and of writing a vCardCollection to a file. To make use of the writing features of the library, see some example code below. 

        
          vCard vcard = new vCard();
          //add some properties
          vcard.Version = 2.1F;
          vcard.Surname = "John";
          vcard.Firstname = "Smith";
          PhoneNumber number  = new PhoneNumber();
          number.Number = "09889302222";
          number.Type = PhoneNumberType.Cell;
          vcard.PhoneNumbers.Add(number);

          string filePath = "path-to-vcf-file";
          //To write a single vcard in its default version
          vcard.Save(filePath, [Optional] WriteOption.ThrowError);
          //To write a single vcard in another version
          vcard.Save(filePath, 3.0F, [Optional] WriteOptions.Overwrite);

          vCardCollection collection = new vCardCollection();
          collection.Add(vcard);
          //To write a vcard collection in their default versions
          collection.Save(filePath, [Optional] WriteOption.ThrowError);
          //To write a vcard collection in another version
          collection.Save(filePath, 3.0F, [Optional] WriteOptions.Overwrite);
